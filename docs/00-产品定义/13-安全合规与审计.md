# 第 13 章 · 安全、合规与审计

> 本章定义系统的 **必备安全能力** 与 **审计日志范围**。是 2.1 / 5.x / 8.x 等章节的横切关注点落地。

---

## 13.1 必备安全能力

| 能力 | 描述 |
|------|------|
| 多因素认证 | 登录需 MFA(短信 / 邮箱 / TOTP) |
| 角色权限 | RBAC(见 4.x) |
| 租户隔离 | TenantScope(见 3.2) |
| 案件权限 | MatterScope(见 5.1) |
| 字段级加密 | L4 / L5 敏感字段加密存储 |
| 文件水印 | 下载 / 预览时打律师 + 时间水印 |
| 下载审批 | L4 / L5 文件下载需主办律师 / 合伙人审批 |
| 导出审批 | 数据导出需审批 + 审计 |
| AI 调用日志 | 10.2 全记录 |
| 客户授权记录 | 隐私授权 / 客户确认全留痕 |
| 操作日志 | 用户所有写操作 |
| 访问日志 | 用户所有读操作 |
| 异常登录提醒 | 异地 / 异常时间登录告警 |
| 敏感案件审批 | L5 案件操作需双重审批 |
| 离职权限回收 | 离职即失效,审计其历史行为 |
| 数据备份与恢复 | 每日全量 + 增量,异地容灾 |

---

## 13.2 审计日志范围

> 所有以下动作均需审计日志留痕(用户 ID / 时间 / IP / 设备 / 输入 / 输出 / 结果),日志不可篡改。

| 类别 | 动作 |
|------|------|
| 账户 | 登录、登出、密码修改、MFA 验证、异常登录 |
| 客户 | 查看客户、修改客户、导出客户数据 |
| 案件 | 查看案件、修改案件、案件状态机流转、案件关闭 |
| 文件 | 查看文件、下载文件、修改文件、删除文件 |
| 数据 | 导出数据、跨租户查询 |
| AI | AI 调用(律时对话 / 文书生成 / 法律研究等) |
| 客户沟通 | 发送客户消息、客户确认风险揭示 |
| 报价 | 生成报价、报价审批、报价发送 |
| 推广 | 内容发布、内容审核、推广账号变更 |
| 线索 | 线索转化、线索状态变更 |
| 任务 | 任务创建、任务修改、任务完成 |
| 文书 | 文书审核、文书下载、文书修改 |
| 知识 | 知识入库、知识访问、知识下载 |

---

> **关键设计要求**
>
> 1. **不可篡改**:审计日志应使用 **追加写** 模式(append-only),定期归档至冷存储
> 2. **可检索**:支持按 `tenant_id` / `user_id` / `matter_id` / `action` / `time_range` 多维度查询
> 3. **可导出**:律所可一键导出审计报告(满足监管检查要求)
> 4. **可回放**:支持"用户某段时间的所有动作"时间线回放
> 5. **告警**:异常行为(高频访问、越权尝试、离职人员访问)实时告警
