# 第 1 章 · 总体判断与设计原则

> 本章给出本系统的根本定位、价值主张与服务对象,后续章节(架构、模块、合规、交付等)均以本章为最高原则。

---

## 1.1 产品不再只是"律师工作台",而是法律服务组织能力系统

优化后的 **LM Unity · Counsel + 律时** 不能只解决"律师如何办案",还必须解决律师行业当前最现实的问题:

1. 律所如何合规获客;
2. 律师如何高效开拓客户;
3. 线索如何合规分诊;
4. 服务如何产品化、透明化;
5. 律师如何避免被销售逻辑绑架;
6. 客户预期如何被正确管理;
7. 案件交付质量如何被持续控制;
8. 客户关系如何沉淀为长期资产;
9. 律师个人经验如何沉淀为组织知识;
10. 律所与独立律师如何在互联网时代建立可信增长机制。

### 关于"网推所"问题

附件文章对"**网推所**"问题的分析非常关键:它不是简单的虚假宣传、非法中介或者冒充律师,而是 **数字化转型、市场扩张和法律服务供给失衡** 共同造成的结构性症候;其中最核心的供给侧问题,是律所长期缺少对案件质量、流程标准、客户预期和执业风险的统一管理能力,导致 **高频、小额、强时效、强情绪负担** 的需求被灰色主体承接。

> 因此,本系统的根本目标 **不是** 帮助律所成为"更高级的网推所",而是帮助律所和律师建立:
>
> **合规 · 可负担 · 可达 · 可信 · 可复制 · 可质控** 的现代法律服务供给体系。

## 1.2 本系统必须同时服务两类用户

本任务书不以某一家律所为唯一对象设计,而应具备通用性,支持:

### A. 律师事务所组织版

| 类型 | 说明 |
|------|------|
| 综合性律所 | 多业务、多团队、多地域 |
| 精品所 | 聚焦特定业务领域 |
| 专业团队 | 律所内某个专业方向 |
| 中小型律所 | 区域型 / 业务型 |
| 区域性律所 | 服务本地客户 |
| 法律服务品牌机构 | 连锁化运营 |

**核心价值 —— "九统一":**

1. 统一获客
2. 统一分诊
3. 统一话术
4. 统一报价
5. 统一风控
6. 统一交付
7. 统一客户成功
8. 统一知识沉淀
9. 统一经营看板

### B. 独立律师 / 小团队专业版

| 类型 | 说明 |
|------|------|
| 独立执业律师 | 个人执业证 |
| 小型律师团队 | 2-5 人协作 |
| 青年律师 | 5 年以下执业 |
| 专业领域律师 | 知识产权 / 涉外 / 婚姻家事等 |
| 个人 IP 律师 | 自媒体、付费咨询 |
| 外部合作律师 | 跨所协办 |

**核心价值:**

- 个人品牌合规管理
- 线索整理
- 客户跟进
- AI 分诊辅助
- 报价卡生成
- 办案任务管理
- 工时记录
- 客户周报
- 知识卡片
- 个人案例库

### 部署模式决策

由上述两类用户并存,系统架构必须采用:

> **多租户 SaaS + 私有化部署 + 个人工作台** 三种兼容模式

| 部署模式 | 主要场景 | 关键特征 |
|----------|----------|----------|
| 多租户 SaaS | 中小所、独立律师 | 共享实例、按租户隔离、快速接入 |
| 私有化部署 | 大型所、涉密业务 | 独立部署、可内网、可深度定制 |
| 个人工作台 | 独立律师 | 轻量、个人数据、社交化协作 |
