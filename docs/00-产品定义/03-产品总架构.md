### 6 层架构的设计逻辑

| 层 | 关注点 | 解决的问题 | 反"网推所"价值 |
|----|--------|------------|----------------|
| **一 · 合规增长层** | 前端:内容 / 线索 / 报价 | 律师如何体面获客 | 可信身份前置 |
| **二 · 客户生命周期层** | 客户:从线索到复购 | 关系资产化 | 责任闭环 |
| **三 · 案件办理层** | 核心:办案全流程 | 律师如何高效办案 | 案件质量可质控 |
| **四 · 律师效率层** | 个人:律师工作台 | 律师个人效率与品牌 | 个人品牌合规 |
| **五 · 质量与风控层** | 横切:贯穿全流程 | 守住 10 条底线 | 全流程合规留痕 |
| **六 · 知识资产层** | 沉淀:从过程到资产 | 经验 → 组织知识 | 知识复用 |

> **关键设计原则**:
> - **层一 / 二 / 三** 是 **主流程** —— Marketing → Sales → Delivery
> - **层四** 是 **个人赋能** —— 主要服务于 B 类用户(独立律师 / 小团队)
> - **层五** 是 **横切关注点** —— Cross-Cutting Concerns,贯穿所有层
> - **层六** 是 **价值沉淀** —— 沉淀自前五层,反哺前五层

### 与 2.3 节 9 大子系统的映射(业务视图 ↔ 工程视图)

| 6 层架构 | 对应 9 大子系统 |
|----------|----------------|
| 一 · 合规增长层 | ① 合规增长中台 |
| 二 · 客户生命周期层 | ③ 客户生命周期系统 |
| 三 · 案件办理层 | ⑤ 案件办理系统 + ⑥ 时间计费系统 |
| 四 · 律师效率层 | ② 律时数字人(对内助理部分) + 独立律师工作台 |
| 五 · 质量与风控层 | ⑦ 质量控制系统 + ⑨ 数据安全与审计系统 |
| 六 · 知识资产层 | ⑧ 知识沉淀系统 |
| 横跨全部 | ④ 法律服务产品库 + ⑨ 数据安全与审计系统 |

> **6 层 vs 9 系统 的关系**
> - **6 层架构** 是"业务视图" —— 按律师工作流自然分层,便于产品和律师理解
> - **9 大子系统** 是"工程视图" —— 按技术能力和数据归属拆分,便于架构师和工程师落地
> - 两者 **不冲突**,前者是"用户视角",后者是"建设视角"

## 3.2 部署模式

### 三类部署模式

> 任务书原文: *系统应支持三类部署模式*。
> 具体三种模式在 [第 1 章 1.2 节](./01-总体判断与设计原则.md) 已定义,此处交叉引用并展开:

| 部署模式 | 主要场景 | 关键特征 |
|----------|----------|----------|
| **多租户 SaaS** | 中小所、独立律师 | 共享实例、按租户隔离、快速接入 |
| **私有化部署** | 大型所、涉密业务 | 独立部署、可内网、可深度定制 |
| **个人工作台** | 独立律师 | 轻量、个人数据、社交化协作 |

### 四级数据隔离

技术上建议采用:

```
TenantID + FirmID + WorkspaceID + UserID
四级数据隔离
```

| 层级 | 含义 | 示例 |
|------|------|------|
| **TenantID** | 顶级租户 | SaaS 平台实例 / 私有化部署实例 / 个人版账户 |
| **FirmID** | 律所 | 某律师事务所 |
| **WorkspaceID** | 工作空间 | 律所内的业务部 / 项目组 / 合规小组 / 个人律师工作室 |
| **UserID** | 用户 | 具体律师 / 助理 / 客服 / 客户 |

#### 隔离策略建议

| 隔离层级 | 隔离强度 | 实现方式 | 适用场景 |
|----------|----------|----------|----------|
| Tenant 隔离 | **强** | 独立数据库 / Schema | 私有化部署、跨所数据合规 |
| Firm 隔离 | **强** | 共享库 + Row-Level Security | SaaS 多律所 |
| Workspace 隔离 | 中 | 共享表 + `firm_id + workspace_id` 复合索引 | 律所内部门 / 项目隔离 |
| User 隔离 | 弱(查询层) | ABAC / 字段级脱敏 | 个人客户池、个人案例库 |

> ⚠️ **个人律师版特殊性**:独立律师(无 Firm)的场景下,FirmID 可退化为个人标识,WorkspaceID 用于"我的案件 / 我的客户"等命名空间。
