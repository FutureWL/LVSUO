### 7.6 内容审查伪代码

```python
def review_marketing_content(content):
    risks = []
    if contains(content, RedlineWords.BLOCKING):
        risks.add("禁止性宣传表达")
    if detect_result_promise(content):
        risks.add("疑似承诺办案结果")
    if detect_win_rate(content):
        risks.add("疑似宣示胜诉率")
    if detect_judicial_relationship(content):
        risks.add("疑似暗示司法关系")
    if detect_low_price_misleading(content):
        risks.add("疑似低价诱导")
    if detect_client_secret(content):
        risks.add("疑似泄露客户信息")
    if risks.has_blocking():
        content.status = "BLOCKED"
        return
    content.status = "LAWYER_REVIEW_REQUIRED"
```

## 模块二:线索与结构化分诊中心

### 8.1 功能目标

将官网、小程序、短视频、搜索、私域、转介绍、老客户等来源的线索统一纳入系统,形成:

**线索识别 → 隐私授权 → 律时接待 → 结构化分诊 → 律师确认 → 产品推荐 → 报价转化 → 后续跟进**

> 附件文章指出,高频法律需求常常不是一开始就需要完整代理,而是需要"**看得懂、用得上、来 得及**"的结构化判断,例如是否有必要走法律程序、证据从哪里补、现在先做哪一步、是否需要律师介入。

## 模块三:法律服务产品中心

### 9.1 功能目标

把律师服务从"只有完整代理"转化为分层产品,适配高频、小额、强时效需求。

> 附件文章指出,大量用户并非需要从立案到执行的全程代理,而是需要 30 分钟结构化诊断、起诉材料清单、律师函、合同审改、庭前准备、单节点代理、简单执行路径评估等中间服务;正规供给长期没有形成"**可买的产品**",导致用户只能在完整代理和无人帮助之间二选一。

### 9.2 产品层级

| 级别 | 名称 |
|------|------|
| P0 | 免费法律信息产品 |
| P1 | 低价结构化诊断产品 |
| P2 | 标准文书产品 |
| P3 | 流程辅导产品 |
| P4 | 单节点代理产品 |
| P5 | 完整代理产品 |
| P6 | 企业订阅 / 常年顾问产品 |
| P7 | 专项法律项目产品 |

### 9.3 标准产品示例

**产品 1:30 分钟结构化诊断**
- 适用场景:客户不确定是否需要律师介入
- 交付成果:诊断纪要、风险提示、材料清单、路径建议
- 交付周期:24 小时内
- 边界:不承诺结果,不代理案件,不出具正式法律意见书
- 升级路径:律师函 / 文书 / 完整代理

**产品 2:律师函**
- 适用场景:合同催收、解除通知、违约通知
- 交付成果:律师函、发送记录、后续建议
- 边界:不承诺对方付款、履约或撤回行为

**产品 3:合同审改**
- 适用场景:买卖、服务、租赁、投资、合作协议
- 交付成果:风险标注版、修改版、审查意见
- 边界:不包含商业谈判,除非另行购买谈判服务

**产品 4:起诉 / 仲裁材料清单**
- 适用场景:客户准备启动程序但证据不清晰
- 交付成果:材料清单、证据缺口、路径提示
- 边界:不代表已经接受完整代理

**产品 5:单节点代理**
- 适用场景:开庭一次、调解一次、证据整理、保全申请、执行线索整理
- 交付成果:节点性服务成果
- 边界:不覆盖其他程序节点

**产品 6:完整代理**
- 适用场景:诉讼、仲裁、执行、重大争议
- 交付成果:委托合同约定的完整代理服务
- 边界:明确阶段、费用、风险和不包含事项

## 模块六:律时数字人系统

| 角色 | 面向 |

| 律时 · 案件协同秘书 | 办案团队 |

### 13.3 客户健康评分

**评分维度:**

| 维度 | 权重 |
|------|------|
| 最近沟通时间 | +15 / -15 |
| 客户问题响应时间 | +10 / -10 |
| 客户重复催问次数 | +10 / -10 |

def calculate_client_health(client_id, matter_id):
    score = 100
    if days_since_last_contact > 7:
        score -= 15
    if unresolved_client_questions > 0:
        score -= 10
    if client_sentiment in ["ANXIOUS", "DISSATISFIED"]:
        score -= 20
    if lawyer_task_overdue:
        score -= 20
    if client_repeated_progress_questions >= 2:
        score -= 10
    if budget_usage > 0.8 and not client_notified:
        score -= 10
    return max(score, 0)
